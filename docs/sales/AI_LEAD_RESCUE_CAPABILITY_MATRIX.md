## What this doc is for

A practical answer guide for prospect conversations. The matrix below tells the operator, for every capability a prospect is likely to ask about:

- whether AI Lead Rescue **can do it today**,
- **how** the current delivery method works (manual / automated / hybrid),
- **how complex** setup is in operator time,
- **what to say in the first pilot** (the actual sentence the operator says on the call),
- the **notes and constraints** the operator must respect,
- and what the **future enhancement** path looks like (if any).

AI Lead Rescue is a **managed lead-response operating workflow with a human operator** — not a chatbot, not an AI agent, not a CRM replacement. Every answer below holds that line. If a capability would force the offer below that line (per `docs/strategy/ABOVE_THE_LINE_STRATEGY_DOCTRINE.md`), the answer is *"not part of the first pilot"* or *"do not promise"*, even when the capability is technically feasible.

## How to read the classification column

| Classification | Meaning |
|---|---|
| **Available now** | Operator can deliver this in the first pilot without further build. Standard pilot scope. |
| **Available manually in first pilot** | Delivered by the operator by hand during the 48-hour setup + 7-day window. No automation promised. |
| **Feasible quick add-on** | Could be added within ~1 week of operator time if a paying pilot requests it. Not standard scope. |
| **Feasible custom build** | Could be delivered as a separately-quoted custom build (multi-week, separate authorisation, separate security review where relevant). |
| **Not part of first pilot** | Out of scope for the first 4 paying pilots; revisit only after the wedge is proven. |
| **Do not promise** | Forbidden by doctrine; do not commit to this even if a prospect asks. |

The classifications stack — a row marked *"Available manually in first pilot"* with a *"Feasible custom build"* future column means *"we do this by hand now, and a future packet could automate it under custom scope"*.

## Doctrine guardrails summarised

Every cell in the matrix above respects the following non-negotiables. If a prospect-side ask would force a breach, the right answer is *"this isn't the right offer for you"* — politely decline.

- **No guaranteed revenue.** Allowed framing per `docs/sales/AI_LEAD_RESCUE_PRICING_GUIDE.md` § 7: *"existing enquiries get captured, alerted, and tracked"*, never *"new revenue"*.
- **No generic AI agency positioning.** AI Lead Rescue is a managed lead-response operating workflow, not a chatbot, not an AI agent, not a CRM, not a sales-team replacement.
- **No spam / bulk outbound.** Warm-network only for first 4 pilots. Cold-scraped lists forbidden.
- **No regulated-data handling without separate security review.** Medical records, financial data, bank statements, minors / children data, regulated industries, customer PII beyond basic lead contact info all require `docs/operations/SECURITY_REVIEW_CHECKLIST.md`.
- **No fake testimonials, fake client logos, fake screenshots.** Pre-proof window — first 4 pilots have no published case studies. Talk about the offer, not invented social proof.
- **No client-facing automated replies sent without operator review.** Translation, chatbot, AI-assisted drafting are operator-side acceleration only. The customer sees operator-reviewed text.

## How to use this matrix on a call

1. Prospect asks *"can you do X?"*. Find the row.
2. Read the **First-pilot answer** column. That's the sentence you say.
3. If the prospect pushes for more detail, refer to the **Notes / constraints** column.
4. If the prospect asks *"can you build a custom version?"*, refer to the **Future enhancement** column.
5. If the matrix says **Do not promise**, decline politely — see the discovery-call script § 9 *Soft close patterns*.

If the prospect asks something not in the matrix, **do not invent an answer**. Tell them you'll come back with a clear yes / no within 1 business day, log the gap as a `note` in the operator cockpit Activity Log (if a lead row exists) or in the prospect list spreadsheet (if it does not yet), and update this matrix in a follow-up docs PR if the question is likely to come up again.
